# **MatrixOne Intelligence 2026 发布报告**

## 2026 年 05 月 25 日

**功能**

- 新增操作日志模块

系统新增操作日志模块，并完成 moi-new 菜单、路由、国际化和模块注册集成，帮助用户追踪关键操作记录，提升平台审计与问题回溯能力。

- 新增数据分享发布 / 订阅能力

数据分享新增发布与订阅能力，支持表级发布选择、发布弹窗、订阅弹窗及相关页面服务接入。用户可将数据发布给其他工作区或订阅可用数据，提升跨工作区数据协作效率。

- Catalog 支持发布 / 订阅状态展示

Catalog 目录树新增“已发布 / 订阅”状态标识，用户可在数据目录中直观看到数据对象的分享状态，减少数据来源和权限状态判断成本。

- Header 新增版本信息下拉

Header 新增版本信息入口，支持在 moi 和 moi-new 中查看前后端构建版本信息，便于运维排查、版本确认和问题定位。

- CloudSigma SSO 能力增强

新增 CloudSigma legacy SSO token 和 bridge 支持，并统一 SSO / CloudSigma API 定义，提升企业身份接入的兼容性与稳定性。

- Explore 接入 RAG / V1 pipeline

Explore 接入 RAG / V1 pipeline，并增强多角色行列权限、SQL pipeline clone replacement、信息提取页码筛选等能力，进一步完善结构化与非结构化数据联合探索体验。

- 新增 workspace 用户数据库连接信息查询 API

新增 workspace 用户数据库连接信息查询 API，方便前端和集成方获取当前工作区下用户可用的数据库连接信息。

**优化**

- Workflow 运行表单校验优化

Workflow 运行表单支持后端下发 `pattern` 后在前端进行校验和错误提示，帮助用户更早发现参数填写问题，减少无效提交。

- 发布 / 订阅交互体验优化

发布和订阅弹窗新增 loading 态、防误关闭、禁用 ESC 关闭和 tooltip 标签展示，降低用户在数据分享过程中的误操作风险。

- 用户创建流程优化

用户创建流程从“新建用户”调整为“邀请用户”，补充说明文案、错误提示和私有化部署场景提示，使账号协作流程更符合实际使用方式。

- Catalog 订阅数据只读体验优化

订阅数据下的写操作统一置灰，并展示只读 tooltip，避免用户对订阅库、表、卷、文件等对象执行不可用操作。

- 数据分享接口接入优化

数据分享接口拆分 publish / subscribe API 类型，并完成 snake_case 到 camelCase 的转换，提升前后端接口使用的一致性。

- 认证模块架构优化

moi-backend auth 模块拆分为 SSO、OAuth、JWT/Authn 等独立包，降低认证模块耦合，提升后续维护和扩展效率。

- Explore RAG 实现优化

Explore RAG 逻辑迁移到 explore package，答案渲染与流程实现进一步拆分，使数据探索链路更清晰，也便于后续能力扩展。

- 工程与依赖优化

CI 检查拆分为按 App 独立触发的 workflow，减少无关检查和资源浪费。E2E 规范补充编写和 mock 使用规则，剪贴板复制能力增加 HTTP 环境 fallback，并升级 LibreOffice 依赖版本。

**错误修复**

- 修复了数据血缘页侧边栏高亮不正确的问题；
- 修复了 HTTP 环境下 `navigator.clipboard` 不可用导致复制失败的问题；
- 修复了 embedding model probe 在 UI 中不可用的问题；
- 修复了 SSO 响应类型、CloudSigma bridge 错误码和 provider 配置解析异常的问题；
- 修复了 Ant Design v6 下 Pagination `size="default"` 非法属性问题；
- 修复了 TanStack router 插件版本不兼容导致镜像构建失败的问题；
- 修复了订阅库下子节点操作按钮仍可点击的问题；
- 修复了 Catalog 节点 ID 重复导致订阅库只读判断失效的问题；
- 修复了文件类型过滤下拉框缺少 JSONL 类型的问题；
- 修复了发布 / 订阅弹窗可能被误关闭的问题；
- 修复了用户邀请场景下 `user not found` 错误提示不友好的问题；
- 修复了 workspace Catalog 表权限继承、租户权限物理授权、对象权限名称持久化和权限恢复链路相关问题；
- 修复了 PBAC / RBAC policy registry、对象权限路由注册和权限对象列表相关问题；
- 修复了 Explore 条件聚合 SQL 提示、多源 SQL 字段展示、query_sql 解析失败上报、SQL resultset 暴露边界和 filter group 结果拆分等问题；
- 修复了 moi-core 文档解析不支持场景 fail-fast、parser API queue 限流、migrator references 加固及 MatrixOne 不兼容 SSO token 索引问题。

## 2026 年 05 月 17 日

**功能**

- 工作流支持自然语言编排决策流

工作流新增 NL2DSL 节点决策流，支持在自然语言生成 DSL 前进行节点确认与调整，帮助用户更可控地完成工作流搭建。同时新增工作流复制能力，便于复用已有流程并快速创建相似任务。

- 作业详情新增拓扑图

作业详情页新增 DSL 拓扑可视化能力，支持查看节点执行关系、下钻子流程并打开节点详情，帮助用户更直观地理解作业执行过程，提升排查与调试效率。

- 新增信息提取与标准 RAG 工作流模板

系统新增内置信息提取工作流模板和标准 RAG 流程模板。信息提取模板可覆盖从数据读取、文档解析、结构化抽取到结果写入的完整链路，降低常见文档处理场景的配置成本。

- 数据探索能力增强

数据探索支持请求级 SQL 用户变量，可在单次请求中灵活传入上下文参数。同时新增 SQL Resultset `resolve_mode` 选择能力，并支持结果集直通模式，在结果集本身即可作为答案时直接返回原始列与行，减少不必要的模型处理。

- 支持 CloudSigma SSO 登录

新增 CloudSigma SSO 对接能力，支持基于 Keycloak JWT Token Exchange 的免登录跳转，优化企业身份集成与登录体验。

- 国际化能力增强

后端新增统一国际化方案，可根据请求语言返回本地化提示信息。工作流相关错误、表单标题、任务摘要和节点展示文案也已接入国际化，提升多语言场景下的使用体验。

**优化**

- 工作流执行状态治理优化

优化工作流执行状态同步机制，通过推送通知与定期校准结合的方式提升作业列表终态刷新准确性，减少作业已结束但页面状态未更新的情况。

- 大型工作流状态存储优化

针对大型 workflow state 进行外置存储优化，降低数据库持久化压力，并兼容旧格式读取，提升复杂工作流执行的稳定性。

- 文档解析质量优化

增强跨页表格续页识别能力，减少跨页表被错误拆分的情况。同时优化 OpenXML 页眉页脚内容合成，确保页眉、页脚中的文本能够进入解析结果。

- 数据探索体验优化

优化 CASE 表达式生成策略，为缺少 `ELSE` 的表达式自动补充兜底逻辑，降低 SQL 执行失败概率。会话标题改为异步生成并提前推送，减少用户等待感。

- 模型资源和运行表单体验优化

模型资源表单新增协议说明、多地址负载均衡提示、下拉搜索和清空能力。运行表单支持自动填充模型默认值，并优化 Copilot 收起行为，减少对画布操作的遮挡。

**错误修复**

- 修复了作业列表刷新后状态不更新的问题；
- 修复了工作流状态过大导致持久化失败的问题；
- 修复了部分文件执行产物无法被正确找到的问题；
- 修复了 Parser / workitem 执行失败时错误信息不够明确的问题；
- 修复了文档解析结果保存时部分文件名上下文缺失的问题；
- 修复了数据资产身份混用导致血缘和结果文件关联异常的问题；
- 修复了结果文件下载后未按预期清理临时文件的问题；
- 修复了数据探索查询数据库时未正确携带用户权限的问题；
- 修复了多个独立单行指标结果无法安全合并的问题；
- 修复了日期列输出中出现 `T/Z` 等格式字符的问题；
- 修复了语义查看大结果集时报错的问题；
- 修复了大文件导入向量写入时连接超时的问题；

- 修复了 LLM / Embedding 模型探测串台的问题；
- 修复了 parser v2 docx 转 PDF 路由无法启用的问题；

## 2026 年 01 月 12 日

**功能**

- PDF 新增公式解析能力

PDF 中的公式现在可以被准确识别并转换为标准 Markdown 公式，支持直接复制和编辑，同时保留原始公式图片用于对照校验，减少公式还原和修改成本。

- 分段功能支持按照层级感知

分段功能支持按照层级感知的方式去进行分段操作，避免把不同标题/章节下的文本放在同一个 chunk 内，保证语义的完整性

**错误修复**

- 修复部分情况下大文件解析可能会被卡住的问题；
- 修复了部分情况下 docx 解析失败的问题；
- 修复了表格中的图片无法解析的问题；
- 修复了文档中的表格没有解析出换行的问题；
- 修复了时序图/流程图的 caption 里信息错误的问题；
- 修复了 nl2sql 步骤序号重复的问题；
- 修复了表格里合并单元格的内容提取时有丢失的问题。
